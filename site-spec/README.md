# Спецификация сайта

Эта папка описывает, каким должен быть сайт в `/site`. Она не содержит пользовательский корпус и не должна становиться частью навигации сайта.

Главный принцип: `/content` остаётся источником содержательного текста, `/site` является текущим построенным сайтом, а `/site-spec` фиксирует требования к структуре и поведению сайта. Если правка в `/site` меняет устойчивое поведение сайта, соответствующее правило нужно обновить здесь.

Файлы спецификации:

- `site-purpose-and-scope.md` — назначение сайта и границы корпуса.
- `content-map.md` — структура сайта и соответствие документов разделам.
- `ui-and-navigation.md` — двухпанельный интерфейс и левая навигация.
- `content-contract.md` — сохранение текста и ролей разделов.
- `links-and-anchors.md` — внутренние ссылки, якоря и внешние ссылки.
- `reading-flow.md` — переходы, `scrollspy`, кнопка следующей главы.
- `acceptance-checks.md` — проверки перед сдачей результата.
