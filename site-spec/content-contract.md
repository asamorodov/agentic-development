# Контракт сохранения содержательного текста

## Сохранение текста

Главный принцип: максимально сохранить текст документов.

Допустимы только технические изменения:

* генерация стабильных якорей;
* добавление alias-якорей, если они нужны для сохранения существующих ссылок;
* техническая адаптация Markdown для отображения;
* обновление левой навигации;
* исправление явно битых ссылок, если цель однозначна;
* удаление мёртвых ссылок, если восстановить цель невозможно.

Не переписывай содержание ради стиля. Не сокращай. Не делай summary. Не превращай живые материалы в карточки. Не добавляй служебный разговор с собой в основной текст сайта. Не вставляй TODO в пользовательский текст.

Перед выпуском сайта нужно проверить, что основной текст исходных документов вставлен без существенных поправок и без суммаризации.

## Что важно не потерять

Сайт должен сохранить характер материалов:

* истории — источник фактуры;
* теория — язык объяснения;
* Cross-Story Synthesis — связка между корпусом и будущим dev-process;
* Handbook — карта режимов;
* Fieldbook — рабочие сцены и артефакты.

Не переставляй эти роли. Не превращай всё в один большой Handbook. Не делай сайт справочником по Codex. Не подменяй истории краткими карточками.
