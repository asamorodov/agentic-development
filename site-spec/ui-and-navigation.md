# Интерфейс и навигация

## Общий UI сайта

Сайт должен быть двухпанельным:

* слева — оглавление и навигация;
* справа — содержимое выбранного раздела;
* границу между панелями можно менять мышкой;
* выбранная глава должна подсвечиваться;
* ширину левой панели желательно сохранять между перезагрузками;
* сайт должен быть удобен для чтения с клавиатурой, мышью и обычными браузерными кнопками назад/вперёд.

Сайт должен корректно отображать Markdown:

* заголовки;
* списки;
* таблицы;
* ссылки;
* inline-code;
* code blocks;
* цитаты;
* Mermaid-диаграммы, если они есть в документах.

Служебные отчёты, README сборки, Markdown-исходники и внутренние build-файлы не должны становиться разделами пользовательского сайта. Их можно создавать как технические файлы, но не включать в основную навигацию и не показывать как часть корпуса.

## Левая навигация: общие правила

Оглавление слева строится по структуре сайта и заголовкам Markdown-документов.

Разделы с подпунктами должны сворачиваться и разворачиваться.

Пункты одного уровня должны быть выровнены по горизонтали. Если у пункта нет подпунктов, на месте стрелки разворачивания должно оставаться пустое место, чтобы текст не смещался влево относительно соседних пунктов того же уровня.

Жирным должны быть пункты первого и второго уровней навигации. Всё ниже второго уровня должно быть без жирного, независимо от того, есть у пункта подпункты или нет. Активный выбранный пункт не должен терять свой уровень жирности: если пункт второго уровня был жирным, он должен остаться жирным и в активном состоянии.

Выбранный пункт должен быть визуально подсвечен.

## Левая навигация: раскрытие по разделам

Основные верхние разделы сайта по умолчанию раскрыты.

### 12 историй

Раздел `12 историй` раскрыт по умолчанию.

`Story_introduction.md` должен быть доступен как начало раздела историй. Его внутренние пункты `Введение к корпусу` и `Как читать корпус историй` должны быть видны в навигации.

Каждая отдельная история должна быть отдельным пунктом внутри раздела `12 историй`.

Внутреннее оглавление каждой истории по умолчанию должно быть свёрнуто. Пользователь должен видеть список 12 историй сразу, а внутренние главы конкретной истории раскрывать по необходимости.

### Теоретическая часть

В `Theoretical_synthesis.md` части I–VIII должны быть отдельным уровнем иерархии в левой навигации. Подразделы внутри частей должны быть вложены внутрь соответствующей части.

Части I–VIII по умолчанию раскрыты.

`Карта источников` должна находиться на том же уровне навигации, что и части I–VIII, а не быть вложенной в последнюю часть. Её подпункты должны быть вложены внутрь неё.

### Cross-Story Synthesis

В Cross-Story Synthesis второй уровень навигации должен быть жирным, как и другие пункты второго уровня сайта.

Третий уровень и ниже — без жирного.

Навигация должна показывать текущую структуру `Cross_story_synthesis.md`, включая новые главы и перенумерованные разделы, если они уже есть в Markdown.

### Handbook

В Handbook главы должны быть видны сразу.

Подтемы внутри глав должны уметь раскрываться и сворачиваться. По умолчанию главы Handbook должны быть свёрнуты: видны сами главы, а подпункты внутри главы раскрываются по необходимости.

### Fieldbook

Сцены Fieldbook должны быть видны как отдельные главы и уметь раскрываться/сворачиваться. По умолчанию сцены Fieldbook свёрнуты.

`Библиотека рабочих артефактов` должна быть отдельным уровнем навигации на одном уровне со сценами. Она по умолчанию раскрыта.

Подпункты внутри библиотеки рабочих артефактов должны быть видны. Если внутри этих подпунктов есть собственные подуровни, они должны быть сворачиваемыми и по умолчанию свёрнутыми.
