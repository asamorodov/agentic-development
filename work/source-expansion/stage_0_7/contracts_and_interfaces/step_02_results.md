# Search step 2 results — data contracts and schema evolution

## Selected sources

- Data contracts — agreements between producers and consumers; include schema, business meaning, quality rules, SLA and behavior.
- Open Data Contract Standard — gives standardized file format direction.
- Schema evolution / schema migration — compatibility, query/application impact, dual writes, backfills and reversible/incremental changes.

## Evaluation

Data contracts and schema migration are not peripheral. They are where AI-generated changes can silently break consumers or data meaning.

## Impact on theory

Add data contract / migration contract to Part XII and security/provenance tail. Also connect to Part V because some contracts are machine-readable specification artifacts.
