# Search step 2 results — compliance as code and auditability

## Источники

- InSpec profiles / controls — executable compliance checks.
- `Compliance as Code: A Study of Linux Distributions and Beyond` — empirical analysis of compliance rules and control mapping.
- Infrastructure-as-code compliance sources — policy checks before infrastructure changes.

## Оценка

Compliance-as-code is relevant for AI-driven SDLC because agents can generate infrastructure and config changes. It turns governance into testable constraints.

## Влияние на план

Policy-as-code belongs with evidence and governance. It should be a supporting artifact, not a new deep case.
