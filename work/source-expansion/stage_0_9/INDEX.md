# Stage 0.9: добор источников для единого замысла

Дата: 2026-06-07  
Режим: кумулятивный архив-оверлей, продолжение Stage 0.5–0.8.  
Статус: источниковый добор после замечания пользователя о деградации русского языка и риске неоднородной коллекции.

В этом цикле цель не в том, чтобы найти ещё больше отдельных тем. После Stage 0.5–0.8 появилось слишком много правильных артефактов: ADR, GSD, BMAD, контракты, трассируемость, SBOM, секреты, наблюдаемость, review capacity. Если просто добавить их в план, документ снова станет каталогом. Stage 0.9 ищет источники, которые помогают связать эти артефакты в единый рабочий замысел.

Пять тем:

1. `flow_and_platform/` — поток ценности, Team Topologies, platform engineering, software catalog.
2. `reproducible_environment/` — воспроизводимая среда выполнения, dev containers, Nix/Bazel, reproducible/attestable builds.
3. `policy_as_code/` — исполняемые ограничения: OPA, policy-as-code, compliance-as-code.
4. `prompt_context_lifecycle/` — жизненный цикл запросов и контекстных артефактов.
5. `artifact_graph/` — граф связей между артефактами: traceability graph, code graph, catalog.

Каждая тема выполнена в три честных поисковых этапа с разными формулировками.
