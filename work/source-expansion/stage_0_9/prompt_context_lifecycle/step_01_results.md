# Search step 1 results — prompt artifacts and prompt evolution

## Источники

- `Prompts as Software Engineering Artifacts` — prompts require systematic development, documentation, maintenance, traceability and reuse; current practice often ad hoc.
- `Prompting in the Wild` — 1,262 prompt changes across 243 repositories; many prompt changes are undocumented in commit messages and may introduce inconsistencies.
- `Promptware Engineering` — frames prompt-enabled systems as needing requirements, design, implementation, testing, debugging, evolution, deployment and monitoring.

## Оценка

This confirms Part III/SPDD direction. But it also warns: prompt artifacts can become a maintenance problem if treated ad hoc.

## Влияние на план

Add prompt/context lifecycle quality to Part III/VI/XII; keep SPDD as stronger answer, not generic prompt management.
