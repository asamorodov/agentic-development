# Search step 2 results — SBOM empirical/tooling problems

## Selected sources

- `A Large Scale Empirical Analysis on the Adherence Gap between Standards and Tools in SBOM` — finds poor tool consistency, license accuracy issues, and compliance limitations across many generated SBOMs.
- `SBOM Generation Tools in the Python Ecosystem` — issues with dependency versions, metadata files, optional dependencies, remote dependencies and ecosystem metadata.
- `Software Bill of Materials in Software Supply Chain Security: A Systematic Literature Review` — adoption barriers include tooling, privacy, standardization, distribution, overhead, exploitability, false positives and hidden packages.
- `State of the SBOM Tool Ecosystems` — comparative maturity and tool ecosystem differences between SPDX and CycloneDX.

## Evaluation

SBOM should not be treated as magic transparency. It is a useful but fragile artifact with tooling and accuracy limits.

## Impact on theory

Part XII should include “SBOM quality / freshness / trustworthiness” as part of lifecycle-tail artifact management.
