# Search step 3 results — supervisory engineering and perception gaps

## Selected sources

- `The Impact of AI Coding Assistants on Software Engineering` — longitudinal study; shift from creation to verification; “supervisory engineering work”; productivity-experience paradox with worsened flow/cognitive load for some participants.
- METR productivity RCT — experienced developers were slower with early-2025 AI tools despite expecting and perceiving speedups.
- AI productivity/value reports — code activity may increase more than final releases or user value.

## Evaluation

AI-driven SDLC must distinguish activity, output, value and human cognitive load. This directly reinforces the anti-catalog / anti-vibe-coding frame.

Use as caution in Introduction/Part I and Part X/XI. Do not make this a deep case unless writing a separate organizational chapter.
