# Search step 2 results — structured agent logging and observability frameworks

## Selected sources

- `AgentTrace` — structured logs across operational, cognitive and contextual surfaces for agent security/accountability/monitoring.
- `AgentSight` — boundary tracing via eBPF, correlating semantic intent and system-level effects; detects prompt injection, loops and bottlenecks.
- Agent observability sources emphasize the gap between high-level intent and low-level actions.

## Evaluation

This strongly supports “agent trace” as an artifact. A safe agentic SDLC needs not only final test output but trajectory observability.
