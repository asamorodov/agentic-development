# Search step 1 results — agentic frameworks / named candidates

## Selected sources

- `From Prompt to Process` taxonomy — important framing source because it explicitly compares frameworks such as Spec Kit, BMAD, GSD, OpenSpec, Spec Kitty, Reversa along specification/context/roles/execution/validation/portability.
- BMAD Method docs — describes AI-driven development framework from ideation/planning through implementation with specialized agents and workflows.
- GSD / Open GSD materials — context-engineering/spec-driven loop and externalized state; older `gsd-build/get-shit-done` seems less authoritative than current Open GSD / GSD Core.
- Open Agent Specification / Agent Spec technical report — declarative specification for agent workflows, with portability/interoperability as main role.

## Evaluation

GSD/BMAD are not the same type as SPDD or Spec Kit. They are **process/framework layer**. They answer “how to run agentic work repeatedly,” not just “how to specify a feature.”

## Impact on theory

Add process/framework layer to Part VI/VII, not to Part V unless comparing with Spec Kit.
