## Смысл изменения

Прежний протокол смешивал три вещи:

1. общий порядок источникового пополнения документа;
2. особенности методологических досье;
3. логику повторных внешних запусков.

Новая структура разделяет эти слои:

- общий протокол отвечает за работу с любым документом, который накапливает материал из источников;
- prompt уточняет этот протокол для методологического досье;
- повторные запуски остаются внешней организацией и не описываются внутри протокола как самостоятельная тема.

## Что делает новый общий протокол

Он задаёт один запуск над явным `{имя документа}`:

1. прочитать языковые, стилевые and source-протоколы;
2. извлечь уже известные первоисточники;
3. извлечь and расширить историю поисковых формулировок;
4. найти новые источники, если это оправдано;
5. прочитать источники and структурно перенести детали в документ;
6. расставить ссылки по месту появления детали and обновить общий список источников;
7. найти кандидаты на иллюстрации;
8. провести русский языковой and стилевой проход;
9. если указан номер прохода, сохранить after/delta/should_stop artifacts.

## Что делает новый prompt по методологическому досье

Он не дублирует общий протокол. Он уточняет, какие детали искать в методологии:

- problem;
- workflow;
- artifacts;
- context;
- roles;
- human gates;
- validation;
- lifecycle tail;
- strengths;
- failure modes;
- comparisons;
- theory vs Handbook / Fieldbook split.

## Почему это лучше для внешнего оркестратора

Внешний оркестратор может многократно запускать один и тот же prompt над одним `{имя документа}`. Протокол не привязан к досье and не содержит лишней бюрократии. Prompt содержит только специфику жанра.
