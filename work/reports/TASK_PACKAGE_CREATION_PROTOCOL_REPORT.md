# Отчёт: спецификация создания пакетов-заданий

Добавлен рабочий протокол `work/protocols/TASK_PACKAGE_CREATION_PROTOCOL.md`.

Протокол фиксирует двухфазную модель подготовки сложных пакетов-заданий:

1. сначала создаются планы целевых групп;
2. затем выбранные группы собираются в исполнительный пакет через Python-gate и opaque record chain.

Ключевые решения:

- целевая группа может содержать один или несколько связанных документов;
- для связанных документов используется `linked-target-edit`, где один prompt правит несколько target files и обязан создать cross-file consistency report;
- один пакет может содержать несколько целевых групп, но группы обрабатываются последовательно;
- план целевой группы состоит из трёх секций: обрабатываемые файлы, файлы для чтения, очередь промптов;
- план целевой группы является build-time документом и не должен попадать в исполнительный архив по умолчанию;
- видимый Python-скрипт остаётся generic runner/materializer и не содержит структуру конкретной задачи;
- payload строится как opaque record chain, а не как таблица `documents × passes`;
- каждый non-final рабочий лист в конце предписывает снова запустить Python;
- финальный рабочий лист не запускает Python повторно, а проверяет и упаковывает результат;
- обязательна emergency packaging command, которая упаковывает все доступные рабочие файлы при обрыве;
- нормальное завершение отличается от обрыва через completion marker.

Дополнительно добавлен шаблон:

```text
work/prompts/TARGET_GROUP_PLAN_TEMPLATE.md
```

Он нужен для ручной подготовки планов целевых групп перед сборкой пакета.
