# SPDD language sanity-pass

Дата: 2026-06-10. Документ: `work/dossiers/SPDD_METHOD_DOSSIER.md`.

Причина отдельной проверки: после общего языкового цикла SPDD был помечен как документ, где один из промежуточных проходов исправлял последствия encoding-glitch. Поэтому SPDD прошёл отдельный sanity-pass после общего терминологического repair-pass.

## Проверено

- Явные mojibake/encoding-глюки в финальном тексте не обнаружены.
- Raw URL сохранены без изменений относительно `git(8).zip`.
- Локальные Markdown targets, уже испорченные предыдущей русификацией, восстановлены: ссылка на историю Jesse Vincent и anchors `#spdd-six-step-workflow`, `#spdd-unit-tests-after-stabilization`.
- Видимый текст вокруг этих anchors исправлен: `Unit tests` / `unit-тесты` в авторском тексте заменены на `модульные тесты`.
- Грубые гибриды вокруг `prompt`, `source-pass`, `business-prompt`, `traceability-risk`, `scope-gate` устранены.

## Исправленные зоны

- Заголовок `Unit-тесты` заменён на `Модульные тесты`.
- `Unit tests в SPDD` заменено на `Модульные тесты в SPDD`.
- `unit-тесты`, `unit-тестами`, `unit-тестов` в обычном тексте заменены на формы `модульные тесты`.
- `бизнес-prompt` заменён на `бизнес-промпт`.
- `источникный срез` заменён на `срез источников`.
- Figure IDs с русско-английскими гибридами `трассируемость-risk` и `область-gate` заменены на более нейтральные `risk-traceability` и `scope-gate`.
- Несколько процессных `шлюзов` в SPDD заменены на `контрольные точки`, чтобы не смешивать процессный gate с API/network gateway.

## Что не менялось

- Не менялись команды OpenSPDD, пути, имена шаблонов и URL.
- Не переводились source titles.
- Не делалась композиционная переработка SPDD-досье.
- Не использовался отклонённый PWG cycle 05.
