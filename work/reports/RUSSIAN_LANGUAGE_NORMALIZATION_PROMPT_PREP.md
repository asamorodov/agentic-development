# Подготовка языкового запуска досье и новых историй

Подготовлены два prompt-файла для запуска отдельного языкового прохода в Codex:

```text
work/prompts/RUSSIAN_LANGUAGE_NORMALIZATION_PROMPT.md
work/prompts/RUN_RUSSIAN_LANGUAGE_NORMALIZATION.md
```

Нижний prompt предназначен для одного прохода над одним документом. Он читает только сам документ и `protocols/rules/russian-language.md`, затем выполняет два шага: языковую нормализацию на русский язык и проверку отсутствия потерь деталей.

Общий prompt запуска предназначен для controller-задачи в Codex. Он не проектирует новый процесс, а использует готовый нижний prompt и существующий `work/automation/run-source-loop.cmd`. Важная техническая поправка: для каждого документа controller должен вычислять `last_pass`, `start_pass = last_pass + 1`, `min_pass = last_pass + 5`, `max_pass = last_pass + 20`, потому что автоматизация считает `--min-pass` и `--max-pass` абсолютными номерами проходов, а не количеством новых проходов текущего запуска.

Scope запуска:

- активные досье из `work/dossiers/`, кроме comparison snapshot;
- три новые истории `content/stories/13_*`, `14_*`, `15_*`;
- без `work/story_dossiers/`, старых 12 историй, отчётов, pass snapshots и теоретических глав.

Стилевой проход намеренно не включён. Стиль, композиция и теоретическая переработка остаются для отдельной будущей операции.
