# Source accumulation protocol simplified

Дата: 2026-06-08  
Статус: переписаны общий протокол и prompt методологического досье после замечаний пользователя.

## Что изменено

### Общий протокол

`work/protocols/SOURCE_ACCUMULATION_DOCUMENT_PROTOCOL.md` теперь короче and less bureaucratic.

Он задаёт направление одного запуска над `{имя документа}`:

- извлечь источники;
- уточнить поиск;
- найти новые первоисточники;
- прочитать источники;
- структурно перенести детали в документ;
- поставить ссылки по месту;
- добавить кандидатов на иллюстрации;
- провести русский языковой проход;
- если задан номер прохода, сохранить artifacts in `passes/`.

Он больше не содержит подробного списка “какие именно детали всегда извлекать”. Вместо этого зафиксировано направление: сохранять угловатость первоисточника and не сглаживать его до общего вывода.

### Prompt методологического досье

`work/prompts/METHODOLOGY_DOSSIER_SOURCE_ACCUMULATION_PROMPT.md` теперь не дублирует шаги протокола.

Он только уточняет жанр:

- методологическое досье — большой consolidation buffer;
- важно понять методологию как рабочий процесс;
- нужно брать workflow, artifacts, context, roles, gates, validation, lifecycle tail, failure modes;
- полноценное сравнение с соседними методологиями убрано из prompt and должно выполняться отдельным проходом.

## Почему так лучше

Протокол теперь пригоден не только для досье, но и для историй, кейсов, source maps and other source-accumulating documents.

Prompt стал жанровым уточнением, а не вторым протоколом.

Это лучше подходит для внешнего оркестратора: он повторяет один запуск, а логика одного запуска остаётся простой.
