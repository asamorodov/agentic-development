# Language and style repair note for plan patch

Дата: 2026-06-07  
Статус: рабочее правило для следующих документов после замечания пользователя о чрезмерном английском.

## Проблема

В Stage 0.5–0.8 рабочих отчётах стало слишком много английского языка. Часть английских терминов неизбежна: названия источников, инструментов, команд, файлов and canonical concepts. Но объяснения, выводы and рекомендации должны быть на русском.

## Правило дальше

1. Английским оставлять:
   - точные имена файлов: `work/discourse.md`;
   - команды: `/spdd-generate`;
   - названия источников: `Agent READMEs`;
   - устойчивые названия инструментов: Open Policy Agent, Backstage, CycloneDX;
   - короткие термины, где перевод хуже: pull request, sandbox, CI.

2. На русский переводить:
   - explanatory prose;
   - recommendations;
   - section titles, unless they are exact source names;
   - descriptions of mechanisms;
   - report conclusions.

3. Не использовать английский как “клей”:
   - не писать “source depth”, если можно “глубина источника”;
   - не писать “workflow layer”, если можно “слой рабочего процесса”;
   - не писать “artifact cluster”, если можно “группа артефактов”;
   - не писать “human gate”, если можно “человеческий gate” только когда gate is a project term; иначе “человеческое подтверждение”.

4. Для финальной теории стиль должен быть суше и инженернее, чем в рабочих отчётах.
