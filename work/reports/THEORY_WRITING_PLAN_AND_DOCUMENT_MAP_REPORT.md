# Отчёт: план несущих узлов и карта рабочих документов

Дата: 2026-06-10.  
База: `git(8).zip` + текущий применённый терминологический repair-pass.

## Что сделано

Созданы два рабочих документа для следующего этапа письма теоретического раздела:

```text
work/theory-writing/CORE_NODES_WRITING_PLAN.md
work/theory-writing/WORKING_DOCUMENTS_MAP.md
```

`CORE_NODES_WRITING_PLAN.md` фиксирует принятый порядок работы: сначала spine-map, затем несущие синтетические узлы, итоговые фрагменты глубоких опорных узлов, переходные мосты, потом пакетное написание глав, composition pass, языковой и стилевой проходы.

`WORKING_DOCUMENTS_MAP.md` группирует рабочие документы репозитория по категориям и объясняет, как ими пользоваться: публичные истории, управляющие документы, ADR, активные досье, story-dossiers, comparative reports, source maps, quality audits, dossier-pass artifacts, языковые документы, протоколы, automation logs, legacy baseline, project docs and overlay service files.

## Важные решения

- Писать теоретический раздел сначала через несущие узлы, а не линейно по главам.
- Для каждой главы использовать уже написанные узлы, досье, истории and story-dossiers.
- Истории остаются factual corpus; теория использует их как якоря, но не пересказывает.
- Технический атлас не заменяет концептуальное объяснение в теории.
- Automation logs и raw SDK outputs не использовать как обычные источники для writing-pass.
