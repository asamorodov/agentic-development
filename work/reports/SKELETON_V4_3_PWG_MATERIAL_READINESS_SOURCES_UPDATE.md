# Skeleton v4.3-PWG material-readiness source routing update

Дата: 2026-06-10  
База: `git(6).zip` + ранее утверждённый `v4.3-PWG` patch.  
Статус: корректирующий проход по рабочему скелетону.

## Запрос

Пользователь попросил убрать из скелетона раздел `Что изменилось относительно v4.2` и расширить раздел `Порядок написания и готовность материала` до всех глав. Для каждой главы нужно указать собранные нами источники, которыми следует пользоваться прежде всего при написании текста.

## Что сделано

Из `work/skeletons/THEORETICAL_SYNTHESIS_REBUILT_SKELETON_V4.md` удалён отдельный changelog-блок `Что изменилось относительно v4.2`. Скелетон теперь читается как актуальная рабочая структура, а не как комментарий к предыдущей редакции.

Раздел `Порядок написания и готовность материала` полностью заменён. Теперь он покрывает:

- общие материалы для всех частей;
- Введение;
- Части I–XIII;
- Заключение;
- Технический атлас.

Для каждой части указаны приоритетные локальные материалы: досье, истории, story-dossiers, сравнительные отчёты, карты источников, старый baseline и quality audits. Отдельно перечислены группы внешних источников, которые уже были собраны и должны использоваться прежде всего при написании соответствующей главы.

## Важные ограничения

1. Это не новый source-expansion pass.
2. Истории по-прежнему остаются первым разделом сайта и не пересказываются внутри теории.
3. Список источников в разделе готовности — не запрет на дополнительные источники, а порядок чтения: сначала использовать уже накопленный корпус, затем раскрывать внешние первоисточники точечно.
4. Отклонённый PWG cycle 05 остаётся неканоничным и не используется как основание теории.
5. Технический атлас сохраняет свою роль: он хранит техническую фактуру, но не заменяет концептуальное объяснение в теории.

## Проверки

- В скелетоне больше нет заголовка `Что изменилось относительно v4.2`.
- Раздел `Порядок написания и готовность материала` покрывает все основные главы скелетона.
- Для `Persistent Work Graph` явно указано использовать только cycles 01–04 и не использовать отклонённый cycle 05.
- Для технического атласа указаны все десять разделов, включая отдельную главу `Persistent Work Graph`.
