# Debug prompt: проверка source accumulation запуска

Этот prompt используется вместо содержательного prompt пополнения, чтобы проверить оркестратор, вложенный `codex exec`, pass-артефакты и `should_stop`.

Он не ищет источники и не меняет реальные исследовательские документы.

## Задача

Используй входные значения, которые добавлены сверху:

```text
{имя документа}
{номер прохода}
{тема}
{используемый prompt}
{run_id}
{worker_id}
```

Нужно:

1. Прочитать или создать `{имя документа}`.
2. Добавить в него раздел `## Debug pass {номер прохода}`.
3. В этом разделе записать: имя документа, номер прохода, тему, используемый prompt, `run_id`, `worker_id`, текущее время, если доступно.
4. Создать обязательные pass-артефакты по путям, указанным в блоке “Обязательные артефакты этого прохода”.
5. В `after_pass` сохранить полный текущий документ после debug-изменения.
6. В `delta` кратко описать, что debug-проход получил на вход и какие файлы изменил.
7. В `should_stop` записать `yes`, если тема содержит строку `DEBUG_STOP_NOW`; во всех остальных случаях записать `no`.

## Ограничения

Не редактируй `work/discourse.md`, общие отчёты, реальные досье или документы других тем.

Финальный ответ в чат должен быть коротким: только статус debug-прохода и пути созданных pass-артефактов.
