# Prompt: языко-стилевая нормализация досье

Это нижний prompt одного прохода многопроходного документного процесса для языко-стилевой нормализации досье.

Правила не дублируются здесь полностью. Перед правкой перечитай и применяй:

```text
protocols/rules/language-style-rules.md
protocols/rules/russian-language.md
protocols/rules/english-source-handling.md
protocols/rules/terminology-and-translation.md
protocols/rules/human-technical-style.md
protocols/rules/content-preservation.md
protocols/rules/source-and-provenance.md
```

## Задача прохода

Работай только с `{имя документа}`.

Нужно последовательно убирать из досье английский связующий язык и приводить текст к русскому стилю по протоколам, не теряя фактуру.

Не ищи новые источники. Не добавляй новые внешние факты. Не превращай досье в главу теории.

## Что исправлять

Переводи и нормализуй обычный авторский текст досье:

- заголовки и подзаголовки, если они не являются точными названиями;
- объяснительные абзацы;
- списки;
- формулировки вроде `workflow`, `source`, `review`, `gate`, `state`, `artifact`, `prompt`, `pass`, `issue`, `context`, `implementation`, `evidence`, `summary`, `notes`, `report`, `risk`, `update`, `task`, `test`, `surface`, `layer`, если они используются как английский клей, а не как точные имена.

Оставляй латиницу, когда это действительно нужно:

- имена инструментов, проектов, файлов, команд, флагов, API, классов, функций, переменных;
- URL и source titles;
- inline code и fenced code blocks;
- slash-команды вроде `/spdd-code-review`;
- устойчивые названия вроде `REASONS Canvas`, `Spec Kit`, `Gas Town`, `Beads`, `Dolt`, `MCP`, если перевод ухудшит точность;
- прямые цитаты и названия источников.

## Анти-деградация

После правки отдельно проверь, что не потеряны:

- ссылки и inline-ссылки на источники;
- URL;
- команды;
- имена файлов;
- названия инструментов и методологий;
- числа, даты, версии, метрики;
- списки источников;
- кандидаты на иллюстрации;
- открытые вопросы;
- важные оговорки, риски, failure modes and constraints.

Если при языковой правке потерялась деталь, немедленно верни её.

## Артефакты прохода

В конце прохода обязательно создай три файла по путям из системного блока:

1. `*_after_pass_NN.md` — полная версия досье после прохода.
2. `*_delta_NN.md` — краткий отчёт о проходе.
3. `*_should_stop_NN.txt` — только `yes` или `no`.

В delta запиши:

```text
## Что переведено / нормализовано
## Что оставлено в латинице и почему
## Анти-деградационная проверка
## Остаточный английский, который ещё нужно обработать
## Решение should_stop
```

## Условия should_stop

Записывай `yes` только если практически весь текст, который по протоколам должен быть русским, уже переведён или нормально переформулирован.

Не ставь `yes`, если ещё видишь заметный английский клей в обычном авторском тексте досье.

Оставшаяся латиница допустима только когда она относится к именам, командам, файлам, коду, URL, источникам, цитатам или устойчивым техническим названиям.

Если сомневаешься, ставь `no` и продолжай чистку на следующем проходе.
