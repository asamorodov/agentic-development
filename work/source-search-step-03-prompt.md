# Шаг 03. Формулировка поискового промпта — Хвост жизненного цикла: сопровождение, долг, security/governance

## Акцент шага

Искать источники, которые не дают рамке AI-driven SDLC стать оптимистическим “AI делает всё быстрее”: сопровождение AI-кода, agentic technical debt, stochastic tax, security before repo, controls inside workflows, OpenSSF/security guidance.

## Поисковый промпт

Найти источники о downstream-части AI-driven SDLC: maintainability, technical debt, code churn, security, governance, supply-chain/security controls, monitoring/audit. Нужны материалы, которые показывают: если AI ускоряет генерацию, надо перестраивать сопровождение, security и governance, иначе bottlenecks и долг смещаются вниз по lifecycle.

Основные запросы:
- AI coding agents SDLC maintenance deployment observability security governance technical debt 2026 arxiv
- agentic coding agents software maintenance technical debt maintainability AI generated code 2026 arxiv
- AI-assisted software development DORA 2025 systems problem AI adoption software delivery performance
- OpenSSF securing agentic AI software development lifecycle 2026


## Критерии включения

- Источник должен помогать именно рамке AI-driven / agentic SDLC, а не просто говорить “AI пишет код”.
- Предпочтение: официальные документы, первичные инженерные отчёты, arXiv/academic papers, крупные research/consulting reports.
- Новости и vendor blogs допускаются только как осторожный фон или источник конкретных терминов/рисков.
- Источник должен иметь понятную функцию в будущей теории: рамка, platform primitive, governance, security, maintenance, contrast.
