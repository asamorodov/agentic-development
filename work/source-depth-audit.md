# Source-depth audit

Purpose: prevent shallow case mentions.

For each important case or source cluster, check:

1. Is the case described with primary-source facts, not only labels?
2. Are mechanisms, artifacts, constraints and failure modes visible?
3. Is the source doing real explanatory work?
4. Could this be shortened without losing anything? If yes, it may be too shallow or not needed.
5. Does the text tell the reader what this source uniquely reveals?
6. Does the section preserve “angular” source texture instead of smoothing everything into generic claims?

Output:

- list of underdeveloped cases;
- targeted repair plan;
- source gaps requiring additional research.
