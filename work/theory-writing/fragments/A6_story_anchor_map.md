## Размещение якорей в основном тексте после второго стилевого прохода

- **Абзацы 2–3**: Mike / Sandvault, Claude Code worktrees, Codex Worktrees и Stripe devbox работают только как фактура для границы исполнения и передачи работы. Из них не выводятся рабочий процесс, право принять результат или Persistent Work Graph.
- **Абзацы 4–5**: HumanLayer, Ronacher, Sandvault endpoints и Quix MCP используются для инструментальной и наблюдательной поверхности: логи, браузер, локальные скрипты, контекстные риски и различение “наблюдение ≠ завершение”.
- **Абзацы 6–8**: Shopify Roast и Quix / Klaus Kode показывают исполняемый рабочий процесс: cogs, phases, state, детерминированный коридор, bounded repeat, повторное воспроизведение и границу “движок процесса ≠ полномочие принять изменение”.
- **Абзацы 9–10**: Stripe Minions остаются якорем платформенного агента: Slack/context → анализатор → devbox → цикл checks/judge/diagnose → пакет PR → человеческое ревью.
- **Абзац 11**: сравнительный узел прямо показывает, что одна история может поддерживать разные слои, если меняется вопрос механизма.
- **Финал**: мост к Persistent Work Graph оставлен без внутренних ссылок: среда исполнения выполняет шаг, а граф работы должен хранить контрольные барьеры, владение, право продолжения, проверяемые свидетельства и завершающую очистку.



## Добавления по глубине источников

- **Roast ↔ Pi:** README показывает, что `agent` cog может использовать `:pi` как provider. Это полезный сквозной якорь: Pi остаётся minimal локальная обвязка, но Roast может вызвать его как узел рабочего процесса.
- **Специфика Quix:** Quix/Klaus Kode сохранён как domain движок рабочего процесса, потому что сам источник ограничивает его конкретным use case и платформой запуска кода.
- **Оговорка по источникам Stripe:** повторное открытие официальных страниц Stripe через извлечение текста не дало основного текста; платформенная механика остаётся привязана к AI Engineer transcript, а официальные страницы — к осторожному фону / метаданные.

## Риски смешения, которые фрагмент должен предотвращать

1. **Sandvault / worktree → рабочий процесс**: изоляция записи и прав не задаёт порядок действий и не хранит смысл работы.
2. **Browser/log/local script → свидетельство**: полезный сенсор или актуатор не равен свидетельству завершения.
3. **MCP → capability without cost**: MCP расширяет доступ, но расходует контекст, добавляет доверенную поверхность и может быть хуже CLI/локального кода.
4. **HumanLayer harness → корпоративная платформа**: хорошая обвязка улучшает траекторию агента, но не создаёт CI, ownership и PR/review path.
5. **Roast / Quix движок рабочего процесса → полномочие**: исполняемый процесс может повторять и восстанавливать шаги, но не присваивает себе право принять изменение.
6. **Stripe platform agent → локальный harness**: Minions используют внутреннюю платформу Stripe; их нельзя переносить как одну команду или один prompt.
7. **Pi minimal harness → universal safety**: локальная изменяемая обвязка полезна внутри доверенной границы, но при другой границе безопасности нужен внешний sandbox/container/user layer.

## Фактура второго прохода по направлениям задания

- **Worktrees:** добавлены детали жизненного цикла Claude и Codex: путь создания, именование веток, файлы, игнорируемые Git, очистка/lock, передний / фоновый контекст и Handoff.
- **Песочница / права:** добавлен модель безопасности Sandvault: отдельная учётная запись, `sandbox-exec`, общая рабочая область, запрет на home/system/mounted drives и последствия `--no-sandbox`.
- **Devbox:** добавлен внешний доклад Mark Doyle как источник под масштаб удалённых сред разработки Stripe и Minion loop.
- **Логи / браузер / локальные скрипты:** усилены Armin и Sandvault: `make tail-log`, ссылка из отладочного письма в stdout, Playwright, CDP endpoint и iOS bridge.
- **MCP:** HumanLayer и Quix теперь вместе показывают не только риск prompt injection, но и цену описаний инструментов в контексте.


## Фактура третьего прохода по направлениям задания

- **Roast как слой рабочего процесса:** добавлено различие `CodingAgent`/`agent` как узла внутри процесса, а не всего процесса; зафиксирована пара “ранняя статья: YAML/prompt-шаги” и “текущий README: Ruby DSL/cogs”.
- **Quix как слой рабочего процесса:** добавлен детерминированный коридор: SDK-запуск Claude Code для узкой задачи, затем кодовый путь upload/run/download-logs, плюс фазовая структура `WorkflowFactory` и сериализованный `WorkflowContext`.
- **Stripe как платформенный слой:** добавлена цепочка анализатор Slack/context → devbox → цикл checks/judge/diagnose → PR + краткое резюме и ссылка на diff в Slack; усилено различие “пакет для ревью” против “ответ в чате”.

## Ограничения широты после второго стилевого прохода

- **Stripe** используется шире остальных якорей, поэтому ограничение вынесено сюда: в основном фрагменте Stripe не является историей “как устроена вся разработка в компании”. Она даёт два механических узла: devbox как границу исполнения и Minion loop как платформенный путь к PR/ревью.
- **Sandvault** не должен расширяться до общей истории автоматизации браузера. В A6 он нужен для границы прав и для примера, где браузер/iOS остаются сенсорами внутри изолированной среды.
- **Ronacher / Pi** не должен становиться “маленькой Stripe”. Логи, Makefile, Playwright и локальные скрипты поддерживают инструментальную поверхность; Pi поддерживает минимальную локальную обвязку, а не корпоративную платформу.
- **Roast и Quix** не должны превращаться в доказательство готовности результата. Они поддерживают исполняемый порядок шагов, повтор и восстановление, но полномочие принять изменение остаётся вне этих механизмов.

## Статус после второго стилевого прохода

Основной текст теперь должен читаться как самостоятельный узел теории: он не требует от читателя знать внутренние досье и не пересказывает истории целиком. Карта якорей остаётся рабочим ограничителем: она фиксирует, где пример поддерживает аргумент, а где его нельзя расширять без нового источника или отдельного раздела.

## Содержательная починка после проверки источников

- Stripe Selective Test Execution больше не стоит как основная цель ссылки для утверждений о метрик ревью. В A6 эта статья остаётся только возможным фоном про CI-субстрат, который платформенный агент наследует.
- Подробная механика Stripe Minion loop оставлена за AI Engineer transcript с явной оговоркой о статусе курируемой публичной страницы. Официальные страницы Stripe используются осторожно: они подтверждают публичный слой Minions / PR / человеческое ревью, но не раскрывают все инженерные детали в извлечённом тексте.
- Внутренние файлы историй остаются навигацией по уже собранным ассетам; в основном фрагменте они не выступают публичными источниками.
