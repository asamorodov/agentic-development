# C2 source usage

## Неиспользованные или сознательно отложенные источники

| Источник / группа | Решение | Причина |
| --- | --- | --- |
| Reversa, OpenSpec, AgentSPEX | Не использованы в основном тексте C2. | Эти источники уже работают в A5 как соседние формы process-as-artifact. C2 должен быть узким мостом от PWG к GSD/BMAD, а не повторять сравнительный синтез A5. |
| Подробные источники `gsd-pi` кроме Auto Mode | Не использованы в основном тексте C2. | Auto Mode нужен как один пример восстановления; остальные детали ведут к техническому атласу или отдельному профилю GSD. |
| BMAD TEA, Document Project, релизные заметки v6.7/v6.8 | Не использованы в основном тексте C2. | TEA и brownfield-documentation сильны, но добавили бы третий слой качества/документирования. C2 держит bridge «роль/фаза → состояние PWG». |
| Подробные Beads/Gas Town эксплуатационные источники за пределами GitHub/Medium/Docs | Отложены. | C2 не должен превращаться в Gas Town-главу; достаточно узкого boundary-якоря о ролях, пространствах и долговечном отслеживании работы. |
| [BMAD Workflow Map Diagram](https://docs.bmad-method.org/workflow-map-diagram.html) | Открыт в source/visual pass, не вставлен inline. | Это внешняя workflow-диаграмма/страница с artifact flow; фактический текст C2 уже покрыт `BMAD workflow map`, а изображение требует отдельного asset-pass/rights/localization решения. |

## Проверка provenance

- В основном фрагменте нет ссылок на внутренние досье, планы или карту документов.
- Все внешние ссылки стоят рядом с утверждениями, которые они поддерживают.
- Факты GSD, BMAD, Beads и Gas Town взяты через первичные публичные источники, указанные в досье; внутренние рабочие документы использованы только для композиционной границы.
- Новые внешние источники поверх перечисленных в разрешённых досье не добавлялись.
- Второй проход добавил источники только под конкретные утверждения о state, gates, role outputs и recovery; третий проход добавил process-as-artifact/Gas Town boundary-фактуру с первичными ссылками; source usage синхронизирован с основным текстом.
- Четвёртый проход не добавлял новых внешних источников: theatre-failure формулировки выведены из уже процитированных GSD/BMAD/Beads фактов и границы PWG из A4.
- Пятый source/depth pass добавил небольшие факты из уже используемых первичных источников GSD/BMAD/Gas Town и открыл BMAD Workflow Map Diagram только как visual candidate; новых широких источников в основной текст не введено.
- Адресный consequence-pass не добавлял источников: он усилил концептуальный критерий, что процессный элемент важен только через изменение состояния работы.
- Адресный process-theater pass не добавлял источников: approval/ceremony формулировки являются локальным развитием уже обоснованного критерия state consequence.
- General editorial pass 1 не добавлял источников: правка ограничилась снятием самоотсылок и русского выравнивания общих терминов в основном тексте.
- General editorial pass 2 не добавлял источников: правка ограничилась публичным слоем вступления, заменой мета-формулы границы, русификацией «процесса как артефакта» в основном тексте и сжатием Gas Town boundary-абзаца без потери фактов.
- General editorial pass 3 не добавлял источников: правка сняла два несущественных терминологических трения в основном тексте (`Quick Flow` как необязательный пример выбора BMAD-потока и generic `lifecycle`), без изменения фактических утверждений.
