## Смысловая карта фрагмента

```text
рабочее наблюдение агента
  → сенсорный след: браузер / DOM / консоль / лог / стенограмма / `replay`
  → рабочая память: почему выбран следующий ход
  → трасса происхождения: откуда взялась команда, вывод, решение или утверждение
  → область действия
  → оракул проверки
  → тип свидетельства: тест / контрактная проверка / архитектурная фитнес-функция / SLO / постепенный выпуск / ревью владельца
  → сохранённая трасса
  → воспроизводимость
  → связь с требованием / контрактом / ADR
  → приемлемый владелец проверки
  → критерий отказа
  → переход состояния в PWG / ADR / ревью / постепенный выпуск
  → свидетельство для человеческого решения
```

Эта карта удерживает фрагмент как теоретический узел. Истории используются как короткие якоря: они дают конкретные рабочие следы, но не заменяют рассуждение о статусе этих следов.

## Заметка второго стилевого прохода

В основном фрагменте истории оставлены как якоря для типов свидетельства, а не как самостоятельные мини-досье. После второго стилевого прохода их роли сужены так:

- Arvid Kahl — только UI-цепочка маршрут/действие/снимок экрана/DOM/консоль/повторная проверка.
- TDAD — источник истины, тестовый оракул проверки и риск подгонки к видимому набору.
- ADR/AWS/Vercel — память решений, `Confirmation`, ревью владельца и связь решения с кодом.
- Shopify Roast — исполняемый журнал и итоговый результат как трасса, но не автоматическое принятие.
- Simon Willison — исследовательский результат, демонстрация, `replay` и стенограмма как смена статуса следа.
- Stripe — независимый проверяющий взгляд, диагностическая обратная связь, PR и человеческое ревью.
- Миграция — оракул паритета, покрытие, критерии переключения; `completed` допустим только после пакета свидетельств.

Если при финальной сборке нужно сократить A7, сокращать следует по деталям внутри этих якорей, а не по самой цепочке артефакт → оракул проверки → владелец → граница завершения.

## Остаточная редакторская заметка

После остаточного прохода истории по-прежнему работают только как проверочные якоря. В основном фрагменте не добавлялись новые сцены; правка была направлена на связность переходов и снятие табличного тона в абзаце о типах свидетельств. Если A7 сокращать, сначала сокращать подробности внутри Simon/Stripe/Migration, но не удалять общий переход от наблюдения к пакету свидетельств.

## Заметка после composition / visual / style repair

A7 после repair-pass ещё сильнее держит истории как якоря, а не как мини-досье. Arvid и Showboat/Rodney получили по одному локальному visual anchor в основном тексте, потому что они прямо показывают смену статуса рабочего следа: browser/runtime observation и демонстрационный документ с сохранённым выводом. Остальные истории оставлены в прозе, чтобы A7 не превратился в каталог проверочных кейсов. При дальнейшем сокращении нельзя удалять общую цепочку: рабочий сигнал → область действия → проверочный критерий → трасса → владелец → переход состояния.
