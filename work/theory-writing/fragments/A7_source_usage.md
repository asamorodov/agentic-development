## Дополнение после усиления линии наблюдений

На шаге усиления линии наблюдений новые внешние источники не добавлялись. Под конкретные утверждения были повторно открыты уже учтённые первоисточники: Arvid Kahl для браузерной линии DOM/консоль, Simon Willison для Showboat/Rodney и `claude-code-transcripts`, Doubrovkine для журналов Roast и `final_output`, AI Engineer Singapore для Stripe проверяющего взгляда с чистым контекстом и диагностического цикла. Эти источники уже присутствовали в таблице; изменены только формулировки использования и места применения.

На шаге усиления линии свидетельств добавлены четыре внешних источника: Pact Docs для контрактной совместимости API-тестов, Thoughtworks Technology Radar для архитектурных фитнес-функций, Google SRE Workbook для SLO и политики бюджета ошибок и Google SRE Workbook для канареечного выпуска. Они добавлены не как обзор новых тем, а как точечные якоря для видов свидетельства, которых не хватало в фрагменте.

На проходе evidence-chain новые внешние источники не добавлялись. Перестройка опиралась на уже введённые источники и внутренние механизмы PWG/ADR: задача была связать наблюдение, артефакт, оракул проверки, владелец и переход состояния, а не добавить новую фактуру.

На проходе source/depth новые внешние источники не добавлялись. Были переоткрыты уже присутствующие источники, уже стоящие в тексте — TDAD A, MADR, AWS ADR process и Vercel `adr-skill`. Добавлены только детали, которые укрепляют уже написанную цепочку: источник истины для тестов, `Confirmation` как проверочный слот, владелец/одобрение коллеги как контрольная точка и связь ADR↔code.

## Публикационный проход проверки источников

Короткий план содержательной починки:

1. Проверить, что ключевые утверждения в основном фрагменте опираются на публичные первоисточники, а не на внутренние досье.
2. Убрать или ослабить формулировки, которые первоисточник не подтверждает буквально. Главная найденная зона риска — фраза про “измеримые ограничения” в Vercel `adr-skill`; источник надёжнее подтверждает план реализации, затронутые пути и проверочные критерии.
3. Не добавлять новые факты; использовать внешний просмотр только для сверки уже стоящих ссылок.
4. Нормализовать сопутствующие карты так, чтобы они не выглядели как самостоятельный публичный аппарат цитирования.

Применённые изменения:

- Повторно сверены основные внешние якоря: TDAD A/B, `tdad-skill`, MADR, AWS ADR process, Vercel `adr-skill`, Pact Docs, Thoughtworks Technology Radar, Google SRE SLO/canary.
- В основном фрагменте исправлена формулировка про Vercel `adr-skill`: вместо неподтверждённых “измеримых ограничений” оставлены план реализации, затронутые пути и проверочные критерии, достаточно конкретные для ручной или программной проверки.
- В `A7_source_usage.md` уточнён статус этой ссылки и отмечено, что внутренние досье использованы только как навигация к первоисточникам.
- Новые публичные источники не добавлялись.
