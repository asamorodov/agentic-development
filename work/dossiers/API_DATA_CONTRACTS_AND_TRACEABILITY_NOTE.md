## Роль в AI-driven SDLC

Контракты и трассируемость закрывают слой, который не покрывается полностью спецификацией, тестами or ADR.

Контракт говорит: какие границы поведения должны сохраняться для других систем, потребителей or данных.  
Трассируемость говорит: как требование, решение, код, тест, выпуск and maintenance связаны между собой.

## Почему это важно для агентов

Агент может внести локально корректное изменение, которое:

- ломает API consumer;
- меняет смысл поля данных;
- нарушает precondition;
- проходит happy-path tests but fails contract;
- делает миграцию невоспроизводимой;
- теряет связь requirement → test.

## Failure modes

1. Contract only as docs, not checked.
2. Traceability as compliance table without real update.
3. LLM-generated code violates implicit contract.
4. Contract tests generated by same model assumptions.
5. Schema evolution ignored until release.
