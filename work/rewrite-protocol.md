# Rewrite protocol

Before rewriting:

1. Read approved design and source precedence.
2. Read old baseline for related material.
3. Read expanded quarry only as material, not as structure.
4. Read relevant case dossiers.
5. Identify governing thesis and lifecycle function.
6. Decide anchor/contrast/drop roles.

During rewriting:

- Preserve old good prose where possible.
- Do not summarize away dense sections.
- Add source links at the point where source-specific facts enter.
- Keep Russian prose clear, literary and technical.
- Avoid headings that are just a list of source/project names, except for deep-case parts where the case itself is the subject.

After rewriting:

- run anti-catalog audit;
- run source-depth audit;
- for SPDD/Gas Town, run anti-degradation audit;
- update ledger and untransferred queue;
- create report and checks.
