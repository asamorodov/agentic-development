# Терминология и перевод английских слов

Примеры слов, которые в обычном объяснении нужно переводить:

tests → тесты;

failures → сбои / провалы / неудачные случаи;

wins → удачные случаи / успехи;

feature → фича / возможность / функция / изменение, по контексту;

setup → рабочая конфигурация / среда / настройка;

default → вариант по умолчанию;

later → позже;

input → входные данные;

output → результат / вывод;

tools → инструменты;

tooling → инструменты / инструментальная среда / инфраструктура;

context → контекст;

maintenance → сопровождение / поддержка / обслуживание;

patterns → паттерны / устойчивые приёмы;

useful → полезный;

when useful → когда это действительно полезно;

works → работает;

stronger model → более сильная модель;

hard thinking → сложное мышление / тяжёлое мышление;

background tasks → фоновые задачи;

issue tracker → трекер задач;

web agents → веб-агенты;

mobile app → мобильное приложение;

desktop app → desktop-приложение или настольное приложение, по контексту;

workflow → рабочий процесс / порядок работы / режим работы;

review → проверка / разбор / ревью, по контексту;

implementation → реализация;

dependency → зависимость;

prompt → запрос, если не имеется в виду точный фрагмент prompt;

summary → резюме / краткое изложение / выжимка;

source → источник;

evidence → свидетельство / подтверждение / проверяемый артефакт;

runtime → среда выполнения / работающее приложение, по контексту.

Примеры плохого результата:

Если задача tricky, он просит вынести всё в spec, отправляет spec в GPT-5-Pro for review через ChatGPT и later вставляет useful parts back.

Лучше:

Если задача сложная или неоднозначная, он просит вынести всё в спецификацию, отправляет её на проверку в GPT-5-Pro через ChatGPT, а затем возвращает полезные идеи обратно в основной контекст.

Плохо:

Людям нужно место, где можно обсуждать реальные failures and wins без marketing layer.

Лучше:

Людям нужно место, где можно честно обсуждать реальные сбои и удачные случаи без маркетингового слоя.

Плохо:

Tests after feature/fix in same context.

Лучше:

Тесты лучше просить сразу после фичи или исправления, пока у агента ещё свежий контекст задачи.

Плохо:

This matters for transfer.

Лучше:

Для переноса этой практики это важная оговорка.
