# Правила перестройки теории

Перестройка теории выполняется в отдельной ветке. Рабочая зона текущей ветки находится в `/work`.

Главный файл задачи:

```text
work/discourse.md
```

Перед любой задачей по перестройке теории сначала прочитайте `work/discourse.md`. Этот файл фиксирует не только итоговые решения, но и ход, через который возникла текущая рамка AI-driven SDLC.

Главные правила:

- Не возвращать expanded theory в статус main draft. Expanded-версия является quarry: источником фактуры, ссылок и наблюдений, но не формой будущего документа.
- Не использовать «карту течений» как верхнюю композиционную рамку вместо AI-driven SDLC. Structural synthesis plan остаётся guardrails.
- Не расширять теорию в основном за счёт новых глав и новых кейсов. Основной акцент — на детальной проработке существующих смысловых зон и переносе фактуры из первоисточников.
- SPDD и Gas Town считать эталонными зонами плотности. Перед изменением этих разделов брать полные версии из baseline, а затем аккуратно адаптировать их к новой структуре.
- Сохранять старую вычитанную прозу, если она совместима с новой архитектурой. Полный перепис допустим только когда старый текст нельзя встроить без разрушения структуры.
- Не добавлять мета-комментарии, рабочие заметки, самодиалог или “план дальнейшей работы” в основной теоретический текст.
- Все внешние детали и источниковые различия вводить сразу со ссылкой на источник.
- Перед написанием глав выполнить обязательные ранние аудиты покрытия: SDLC artifact coverage и agentic frameworks coverage.

## Рабочие источники в `/work`

Для задач по теории обычно релевантны:

```text
work/approved-ai-sdlc-plan.md
work/source-precedence.md
work/baseline-restore-rule.md
work/anchor-case-baseline-restore-rule.md
work/theory-source-map-ai-driven-sdlc.md
work/old-site-theoretical-synthesis-baseline.md
work/expanded-quarry-theoretical-synthesis.md
work/anchor-seed-spdd-old-site.md
work/anchor-seed-gas-town-old-site.md
```

Codex должен сам оценить, какие из них нужны для текущего шага, но не должен пропускать `work/discourse.md`.

## SPDD и Gas Town

Для SPDD и Gas Town действует hard baseline restore rule: сначала использовать старые версии из сайта целиком как baseline/seed, затем адаптировать и дополнять под новую AI-driven SDLC структуру. Нельзя сжимать или снижать детализацию этих разделов без human gate.

## Завершение работы

После нетривиальной thread-сессии по перестройке теории обновите `work/discourse.md`. Обновление должно сохранять смысловой ход, причины решений и новые ограничения, а не только статус выполнения.
