# Skill: codex-task-work

## Когда применять

Применяйте этот skill для нетривиальной работы Codex с текущей задачей в репозитории: подготовка аудита, правка протоколов, изменение сайта, работа над теорией, перенос решений, проверка структуры, создание отчётов или выполнение staged-задачи.

Этот skill не предназначен для ChatGPT-write-through-GitHub режима. Для него используйте `protocols/skills/chat-github-repo-work.md`.

## Что прочитать

Перед началом прочитайте:

```text
AGENTS.md
project/repository-structure.md
project/source-precedence.md
protocols/rules/codex-task-work-protocol.md
```

Если работа идёт в рабочей ветке с `/work`, также прочитайте:

```text
work/discourse.md
```

Затем оцените сопутствующие документы в корне `/work` и прочитайте те, которые нужны для операции.

## Как работать

Следуйте `protocols/rules/codex-task-work-protocol.md`.

Сначала восстановите текущий ход задачи из `work/discourse.md`. Перед правками объявите scope, риск, файлы, которые будут затронуты, и проверки. Для рискованных операций сначала сделайте planning-only pass.

Выполняйте минимальный связный шаг. Не смешивайте unrelated изменения.

## Завершение

После нетривиальной Codex-сессии обновите `work/discourse.md` по `protocols/rules/discourse-maintenance-rules.md`. Если принято устойчивое проектное решение, обновите `project/decisions.md`. Если изменилась общая модель работы репозитория, обновите соответствующий файл в `/project` или `/protocols`.

## Итог

В финальном ответе укажите изменённые файлы, прочитанные правила и источники, выполненные проверки, невыполненные проверки, как обновлён дискурс и какие вопросы требуют решения человека.
